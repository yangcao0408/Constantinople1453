Production Notes

How this face design becomes an importable Tabletop Simulator custom deck.
Card proportions
5:7 (poker size). This mockup is 300×420px on screen — export finals at 750×1050px per card (300 dpi at 2.5×3.5in) so text stays crisp when TTS players zoom in.
Deck sheet layout
TTS's Custom Deck tool reads one image as a grid of up to 10 columns × 7 rows (70 cards). Fill left-to-right, top-to-bottom; leave trailing unused cells blank rather than reordering.
Sheet resolution
750×1050 per cell × a 10×7 grid = 7500×7350px, over most engines' texture ceiling. For a ~35–45 card deck, use a 7×7 grid at 640×896px per cell (4480×6272px) instead — comfortably under 4096 per side once you drop to 8×8 or split the deck into two sheets.
Art panel
The colored scenes are placeholders, not fixed art — the .art box is just a 252×132 (≈1.9:1) frame with overflow:hidden. Swap its contents for <img src="..." style="width:100%;height:100%;object-fit:cover"> and any photo or illustration will fill it, auto-cropped to fit — landscape images near 1.9:1 crop the least.
Card back
One shared back (bottom-right sample above) — TTS uses a single back image for the whole deck unless "unique back per card" is enabled, which this deck doesn't need.
  1. Finalize the face art. Duplicate this HTML per card (or template it — swap .card-title, .event-text, ops-badge, and the .art variant), one file or canvas per card.
  2. Rasterize each card at 750×1050 (a headless browser screenshot, or open each in a browser and capture the .card element's bounding box).
  3. Composite into a grid sheet — 7 columns is a clean fit for a 35–45 card Event Deck across one or two sheets. Any image editor's contact-sheet / batch tool works; keep cell order matching the card numbering in the corner (EV·01, EV·02...).
  4. In Tabletop Simulator: Objects → Components → Custom → Deck. Set Face to the sheet, Back to the single back image, and enter the deck's row/column count and total card count to match your sheet.
